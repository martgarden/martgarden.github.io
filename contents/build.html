Statické stránky na githubu generované z šablonového souboru "mooster.html" do kterého jsou perlovým scriptem vloženy všechny stránky v adresáři "contents".

  • Cílové soubory jsou otevřeny všechny naráz, takže čteme "mooster.html" jen jednou.
  • Místo pro vložení označeno tagem "".
  • Lokalisace:
    • V jednom souboru jsou uloženy všechny jeho jazykové verse.
    • Neoznačený obsah uložen do všech jazykových verzí
    • Obsah mezi jazykovými tagy (např. "english text") uložen pouze do příslušné verse.
    • Všechny verse se opět tvoří dohromady (filehandly uloženy v hashi klíčoveném jazykovými zkratkami).
    • Text "" nahrazen názvem souboru. To umožňuje přepínat mezi různými jazykovými versemi stejné stránky (použito v záhlaví).
  • Deploy pomocí vlastní github-workflow.

Static website on github generated from the template file "mooster.html". Into this file are one-by-one inserted the files in the "contents" folder.

  • The target files are all opened simultaneously, hence "mooster.html" can be read only once.
  • The place for insertion denoted by "
  • Localisation:
    • One file contains all of its language versions.
    • Unmarked content is inserted into all the language versions.
    • Content between the language tags (e. g. "english text") is inserted only in the proper language version.
    • All the language versions are again created at the same time to avoid reading the files multiple times.
    • Text "" replaced by the name of the file. This allows me to reference from one language version of a page to different one (used in the header).
  • Deployed via custom github-workflow.
